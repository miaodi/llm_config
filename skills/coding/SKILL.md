# Coding Skill

## Purpose
Support implementation, debugging, refactoring, and review tasks.

## When To Use
Use for repository changes, code explanations, or defect investigation.

## Workflow
1. Inspect the relevant code before proposing changes.
2. Minimize surface area and preserve existing behavior unless asked otherwise.
3. Verify with tests or targeted checks where practical.

## Output
Code changes, findings, and any verification notes.
